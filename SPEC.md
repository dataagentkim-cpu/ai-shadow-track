# AI 투자 판단 섀도우 트랙 — Claude Code 빌드 브리프

## 한 줄 정의
국내주식(코스피+코스닥)을 대상으로, AI가 매주 백지에서 포트폴리오 판단을 내리고 그 **논리와 확신도를 로그로 축적**한 뒤, 실제 시장 결과 및 나의 실제 보유 성과와 대조해 "AI 투자 판단이 나보다 나은가"를 데이터로 검증하는 개인용 섀도우 트랙 시스템.

## 목적 (이걸로 진짜 알고 싶은 것)
- AI가 종목을 고르는 판단이 실제로 실력인가, 아니면 운·서사인가.
- 그 판단이 **나 자신의 판단보다** 나은가.
- 부수 목적: 이 "판단 로그 → 사후 검증" 루프를 개인 돈으로 먼저 돌려보고, 회사 일반계정 섀도우 트랙 설계의 프로토타입으로 삼는다.

## 절대 원칙 (설계 전반을 지배)
1. **돈을 굴리지 않는다.** V1은 페이퍼 트래킹. 실매매 없음. 신뢰가 트랙레코드로 쌓인 뒤에야 다음 단계를 논한다.
2. **로그가 결과보다 중요하다.** 수익률만 비교하면 그냥 게임이다. "왜 이 판단을 했는가 / 확신도"를 남겨야 3개월 뒤 *틀렸을 때의 논리*를 해부할 수 있다.
3. **깔때기로 좁힌다.** 전체 시장은 후보 자격은 유지하되, LLM에 2,700개를 다 밀어넣지 않는다. 정량 스크리너가 먼저 shortlist로 좁히고 → 그 shortlist만 LLM이 깊게 추론.
4. **블라인드.** AI는 나의 현재 보유 종목을 **모른 채** 판단한다. 순수 대결을 위해 보유 데이터는 벤치마크 계산에만 쓰고 AI 판단 프롬프트에는 절대 넣지 않는다. (데이터 격리 필수)
5. **오늘이 출발선.** 세 트랙 모두 오늘 스냅샷 기준으로 시작한다.

## 벤치마크 (다트랙, C-라이트: 3렌즈 병렬)
매주 아래 트랙들의 총수익을 나란히 추적한다. 전부 오늘부터 시작.

| 트랙 | 정의 | 논리 로그 |
|------|------|-----------|
| ① 내 보유 홀딩 | 오늘 시점 나의 실제 보유 종목·비중을 그대로 홀딩 | 시작 시 1회 (내가 왜 들고 있는지) |
| ②a 가치-AI | 가치 렌즈 고정, stateful, ①엔 블라인드 | 매주 |
| ②b 모멘텀-AI | 모멘텀 렌즈 고정, stateful, ①엔 블라인드 | 매주 |
| ②c 퀄리티-AI | 퀄리티 렌즈 고정, stateful, ①엔 블라인드 | 매주 |
| ②d 자유형-AI | 고정 렌즈 없음. 자율 판단하되 **매주 어떤 관점으로 골랐는지 선언**. stateful, ①엔 블라인드 | 매주 (관점 선언 포함) |
| ③ 지수 홀딩 | 코스피(+코스닥) 지수를 그냥 홀딩 (총수익지수 기준) | 없음 (정직한 하한선) |
| ④ 스크리너 동일가중 | 매주 shortlist를 동일가중, **LLM 없이** 보유. ②와 동일 체결·비용 규칙 | 없음 (렌즈 없음 공통 대조군) |

**핵심 기준** — 세 질문을 분리:
- (a) **"AI가 나보다 나았나"** = ②x vs ① (진짜 목적)
- (b) **"각 렌즈가 스크리너 위에 값을 하나"** = ②x vs ④. 못 이기면 그 렌즈의 LLM은 값을 못 한 것.
- (c) **"어떤 렌즈가 이 시장에서 통했나, 그리고 고정이 나은가 자유가 나은가"** = ②a vs ②b vs ②c vs ②d. 준승이 뭐가 맞는지 정할 필요 없이 시장이 답한다. ②d(자유형)의 관점 선언 로그를 읽으면 "AI가 제약 없을 때 어떤 렌즈로 수렴하는가"까지 관찰 가능.

### 벤치마크 ① 실제 보유 (오늘 스냅샷, 평가금액 기준)
`holdings` 테이블 시드. **AI 판단 프롬프트에는 절대 미투입 (블라인드).** 전방 수익률은 현재가·평가금액 기준으로 추적한다.

| 한화오션 | 132 | 83,300 | 10,971,194 | 19.2% |
| 삼성중공업 | 413 | 22,050 | 9,087,072 | 15.9% |
| 이오테크닉스 | 25 | 341,000 | 8,506,672 | 14.9% |

## 시스템 아키텍처 (주간 사이클)

```
[1] 데이터 수집 (주말 스냅샷)
     코스피+코스닥 전 종목 가격·거래량·시총·기본 재무
          ↓
[2] 정량 스크리너 (룰 기반, LLM 아님)  ← 상세는 아래 "스크리너 설계"
     1차 정량 필터 → 기계적 중복 제거 → 클러스터 다양화 → 수십 개 shortlist
          ↓
[3] 뉴스·심층 데이터 수집
     shortlist 종목 + 시장 전반 뉴스만 (전체 아님)
          ↓
[4] LLM 판단 (Claude, 블라인드)
     shortlist + 데이터 → 목표 포트폴리오 + 종목별 논리 + 확신도
          ↓
[5] 로그 기록 (DB)
     AI 판단 저장. 3파전 수익률 스냅샷 저장.
          ↓
[6] 사후 검증 (누적)
     지난 판단들을 이후 시장 결과와 대조. AI/나 각각의 적중·오판 논리 해부.
```

**실시간 아님**: 주 1회 주기에는 실시간 데이터가 불필요하다. 주말 종가/거래량 스냅샷으로 충분하며, 실시간은 인프라만 복잡하게 만든다.

## 스크리너 설계 (깔때기 상세)

**대원칙 — 입력은 깨끗하게, 집중은 AI의 선택으로.** 스크리너는 shortlist가 기계적으로 중복되거나 한 방향에 쏠리지 않게 *정리*하는 데까지만 한다. 특정 섹터에 몰빵할지는 AI가 논리·확신도와 함께 내리는 결정으로 남긴다. **스크리너가 강제 분산하지 않는다** — 그러면 나중에 "AI가 판단해서 집중한 건지, 스크리너가 막아서 못 한 건지"를 로그로 구분할 수 없어 실험의 정직성이 깨진다.

3단계로 좁힌다:

1. **1차 정량 필터**: 거래대금·시총 하한, 관리종목·거래정지·스팩 제외 등 기본 위생 + 모멘텀/밸류에이션 랭킹.
2. **기계적 중복 제거 (P0, 하드)**: 보통주/우선주 쌍 → 하나만. 동일 발행사·지주-자회사 중복 → 대표 하나만. (사실상 같은 베팅이므로 무조건 제거)
3. **클러스터 다양화 (쏠림 정리)**: 다른 회사지만 같이 움직이는 묶음(은행·2차전지 소재 등)이 shortlist를 도배하지 않게 대표만 올린다.
   - **V1 (간단)**: 업종 코드 기준, 업종당 최대 N개(예: 3)까지만 shortlist 허용.
   - **업그레이드**: 최근 60~120일 수익률 상관관계로 클러스터링 → 클러스터당 대표 종목만. (더 정교하지만 구현 부담 큼)

> 다시 강조: 2·3단계는 *shortlist 입력*을 정리하는 것이지, AI가 관련주에 집중하는 걸 금지하는 게 아니다. AI가 한 섹터 3종목을 담고 싶으면 담되 논리·확신도를 남긴다.

## 데이터 소스 (Claude Code가 구현 시 가용성 검증할 것)
- **가격/거래량/시총/재무 (KRX)**: `pykrx`, `FinanceDataReader` — 국내 시장 표준 파이썬 라이브러리. 코스피+코스닥 전 종목 스냅샷 가능.
- **지수**: 코스피/코스닥 지수도 위 라이브러리로 취득.
- **뉴스**: 네이버 뉴스 등. shortlist 종목 + 시장 전반만. 전체 종목 뉴스는 수집하지 않는다.
- 각 라이브러리/엔드포인트가 실제로 동작하는지 빌드 시점에 최소 예제로 먼저 확인하고 넘어갈 것.

## LLM 판단 규칙
- **주기**: 주 1회, **월요일 장 마감 후** 판단 (config로 빼둘 것 — 격주/월간 비교 가능하게). 매일은 노이즈.
- **모델**: `claude-sonnet-5` 권장(주간 배치라 비용·속도 적정). *모델 문자열은 변경될 수 있으니 빌드 시점에 Anthropic docs에서 현재 문자열 확인.* 최종 shortlist에 대한 판단만 더 강한 추론이 필요하면 그 단계에만 Opus를 쓰는 것도 옵션.
- **인풋 (stateful, 단 ①엔 블라인드)**: shortlist 종목의 가격·거래량·재무·뉴스 + 시장 전반 컨텍스트 + **지난주 AI 자기 포트폴리오**(decisions 최신 회차). 두 종류의 블라인드를 구분할 것:
  - AI는 **자기 과거 포트는 안다** (stateful).
  - AI는 **나의 실제 보유(① 트랙)는 절대 모른다** (블라인드 유지). ①은 벤치마크 계산에만 쓰고 프롬프트에 미투입.
- **stateful 프레이밍(처분효과 방지)**: 지난주 보유 종목을 줄 때 **매수가·현재 수익률은 가리고**, "오늘 이 가격에 새로 산다면 살 것인가?"를 기준으로 hold/trim/add/sell 판단하게 한다. **'전량 유지(거래 없음)'도 유효한 출력**으로 허용 — 가만히 있기가 선택 가능해야 처분효과 가설을 검증할 수 있다.
- **아웃풋 포맷** (모든 렌즈·나 공통 로그 스키마):
  - `종목`
  - `액션` (매수 / 매도 / 홀딩)
  - `목표비중` (%)
  - `논리` (왜 이 판단인가, 자연어)
  - `확신도` (상 / 중 / 하)
- **각 렌즈는 독립 시스템 프롬프트 + 독립 stateful 포트**를 가진다(②a/②b/②c 각각 자기 과거 포트만 참조). 셋 다 같은 shortlist를 받되 자기 렌즈로만 판단.
- **temperature 낮게 고정**(예: 0.2~0.3). 주 단위 판단 변화가 무작위가 아니라 렌즈·시장에서 나오도록. 가끔 같은 인풋 2~3회 돌려 흔들림을 로그에 남기면 판단 안정성 점검 가능.
- 나의 판단(①)도 시작 시 같은 포맷으로 로그.

## 투자 렌즈 정의 (C-라이트: 3렌즈 병렬)
각 렌즈는 **고정된 투자 철학**으로 매주 일관되게 판단한다(렌즈를 매주 바꾸지 않는다 — 그래야 "그 렌즈가 통했나"를 검증 가능). 준승이 뭐가 옳은지 정할 필요 없이, 셋을 병렬로 돌려 시장이 답하게 한다. **스크리너는 렌즈 중립**(유동성·시총 필터 + 중복 제거만, 팩터 편향 없음) — 렌즈 차이는 전적으로 LLM 선택 단계에서만 발생.

**②a 가치 (Value)** — 펀더멘털 대비 싼 것을 사서 기다린다.
- 산다: 저PER·저PBR·저EV/EBITDA, 높은 이익·배당수익률. 시장이 과도하게 비관한 종목.
- 판다: 적정가치 도달(더 이상 안 쌈), "싸다"는 근거 훼손, 더 싼 대안 발견.
- 사이징/확신: 안전마진 클수록 큰 비중. 애매하면 현금. 회전 낮음, 역발상 허용.

**②b 모멘텀 (Momentum)** — 오르는 것을 사서 추세를 탄다.
- 산다: 3~12개월 상대강도 상위, 신고가 부근, 실적·수급 모멘텀.
- 판다: 추세 이탈(이평 하향 돌파)·상대강도 약화. **이기는 종목은 오래 들고, 지는 종목은 빨리 자른다.**
- 사이징/확신: 추세 강할수록 큰 비중, 반전 시 신속 축소. 회전 중간, 손절 규율 강함.

**②c 퀄리티 (Quality)** — 좋은 회사를 적정가에.
- 산다: 높은 ROE·안정적 이익성장·낮은 부채·높은 마진·꾸준한 현금흐름.
- 판다: 퀄리티 훼손(ROE 하락, 부채 급증, 마진 압박), 밸류에이션 비정상 과열.
- 사이징/확신: 퀄리티 확실할수록 집중, 미검증 기업 회피. 장기보유·우량주 편향, 회전 낮음.

**②d 자유형 (Free)** — 고정 철학 없이 자율 판단.
- 산다/판다: AI가 매주 스스로 최선의 관점을 골라 판단. 고정 렌즈 강제 없음.
- **필수**: 매주 로그에 **"이번 주 어떤 관점(밸류/모멘텀/퀄리티/뉴스/기타)으로 골랐는지"를 명시 선언.** 이 선언이 이 트랙의 핵심 산출물 — 수익률보다 "AI가 제약 없을 때 어디로 수렴하는가"를 보는 게 목적.
- 나머지 규칙(stateful, ①엔 블라인드, "오늘 새로 살까" 프레이밍, 델타 비용)은 다른 렌즈와 동일.
- 해석 주의: ②d가 고정 렌즈를 이겨도 표본이 작으면 운일 수 있음 — 몇 년 단위로 볼 것.

## 체결 시점 규칙 (look-ahead 편향 방지 — 반드시 지킬 것)
**판단에 쓴 데이터보다 체결은 항상 나중 시점이어야 한다.** 판단 시점 = 체결 시점이면, AI가 "오르겠다"는 정보를 다 본 상태로 오르기 전 가격에 사는 게 되어 성적이 구조적으로 부풀려진다.

확정 타임라인 (B안):
- **판단**: **월요일 종가**/거래량/뉴스 기준 (월요일 장 마감 후 배치 실행) → 로그 기록.
- **체결**: **화요일 시가(개장가)**로 가정. 판단과 체결 사이 하루 간격 강제 → 갭상승/하락 자동 반영.
- **슬리피지**: 체결가에 불리한 방향으로 **0.3%** (매수 +, 매도 −).
- **거래비용**: 매도 시 **증권거래세(2026년 기준 매도금액 0.20%) + 브로커리지 수수료(0.015%, 양방향)**를 반영. **비용은 지난주 대비 실제 거래된 델타에만 부과** — 유지되는 포지션엔 비용 없음(전체 회전에 물리지 말 것). 리밸런싱이 잦을수록 이게 수익률을 갉아먹어야 정직하다.
- **네 트랙 통일**: ①②③④ 전부 **화요일 시가**를 출발·체결 기준으로 맞춰 출발선을 통일한다.
- 즉: `월요일 종가 판단·로그 → 화요일 시가 체결(+0.3% 슬리피지+거래세·수수료) → 그 시점부터 수익률 추적`.

## 평가 방법론 (프로 수준 격리·정직성)

**A. LLM 기여도 격리** — ④ 스크리너 동일가중 트랙으로 "LLM이 값을 하는가"를 격리. ②−④ 스프레드를 매주·누적 기록. ②가 ④를 못 이기면 정량 필터가 다 한 것.

**B. 총수익(배당 포함)** — 모든 트랙을 배당 재투자 총수익으로 계산(수정주가 또는 명시적 배당). 체결가는 raw open 유지, 보유기간 배당은 수익률에 가산(현금배당 시 배당소득세 15.4% 차감). ③은 총수익지수(TR) 기준.

**C. Point-in-time 무결성** — LLM에 들어가는 모든 데이터(뉴스·재무·가격)는 판단 시점(월요일 종가) 이전 공개분만. 뉴스는 게시일시 ≤ 판단시각 필터, 재무는 판단일 이전 공시분만. 보유 종목 거래정지·상폐 처리 규칙 정의(예: 마지막 체결가 청산), 편입/이탈 이력 로깅(생존편향 방지).

**D. 위험조정 지표** — 트랙별 매주·누적으로 수익률·연율화 변동성·Sharpe·Sortino·MDD·적중률·회전율 기록. ②·④의 ③ 대비 **베타**와 **잔차수익(알파)**을 별도 컬럼으로, 섹터 편중도 매주 기록. "이긴 게 실력(알파)인지 위험을 더 진 것(베타)인지" 분리.

**해석 규율 (코드 아님, 사람이 지킬 것):**
- **표본 크기**: 주 1회 = 연 52판단. 실력/운 구분엔 **몇 년**이 필요. 3개월~1년 결과는 사실상 노이즈 — 초반에 승부 선언 금지.
- **과최적화 방지**: 결과 보고 스크리너·프롬프트·유니버스를 자꾸 고치면 그 데이터에 과최적화됨. "이 버전으로 N개월 안 건드림"을 정하고 지키는 것 자체가 실력. 전진 섀도우의 out-of-sample 순수성이 최대 강점이니 오염시키지 말 것.
- **겸손한 사전확률**: 비용 차감 후 능동 전략 대다수는 장기적으로 지수를 못 이김. 귀무가설 = "AI는 지수를 못 이긴다", 이긴다는 증거 문턱을 높게.

  - `universe_snapshot` / `screener_output` (주간 후보군 기록 — 사후에 "왜 이 종목이 후보에 올랐나" 추적용)
